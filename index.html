Data Factory Column Mapping

Azure SQL Database Source Dataset can have 4 columns in the structure, but the source database table has 6 columns. Azure SQL Database Source Dataset structure map with source database table column by the column name, so the column order in structure doesn’t matter.

Column mapping supports the following scenarios:

  • All columns in the source dataset structure are mapped to all columns in the sink dataset structure.
  • A subset of the columns in the source dataset structure is mapped to all columns in the sink dataset structure.

The following are error conditions that result in an exception:

  • Either fewer columns or more columns in the “structure” of sink table than specified in the mapping.
  • Duplicate mapping.
  • SQL query result does not have a column name that is specified in the mapping.

From https://docs.microsoft.com/en-us/azure/data-factory/data-factory-map-columns

Cannot Rerun the activity from Data Factory Monitor

Sometime we cannot rerun the activity from the Data Factory Monitor, we click rerun, then refresh the page, the activity status is still showing “Waiting”. It looks like a bug on the data factory monitor.

In this case, we can go back to the Data Factory home page, go to Contents->Pipelines, select the pipeline and check the activity dataset,
drill down to the status of the slice of the dataset, then click run or valid to verify the status. it will start the actvity.

Mockaroo

Mockaroo type VS SQL Server Type

Basing on the SQL Server Data Type to create the Mockaroo schema script, and import the schema file into Mockaroo to generate the test data. You need to sign up an account in the Mockaroo site, then you can import the schema.

We get all the SQL Server data type from system table sys.types, map the sql server data type into Mockaroo type.

Below is the mapping for SQL Server data type and Mockaroo Schema Definition.
value in <> will be replaced by the column definition from system table sys.columns.

Store Procedure to Generate Mockaroo Schema

Store Procedure as below to generate the Mockaroo Schema Definition, then copy the result save it as a json file. Import it into the Mockaroo. Then we can generate the testing data. In the store procedure, I replaced the original column name, in case of the sensitive information was sent out.

CREATE PROC [tst].[sp_GenerateMockarooSchema] @pTableName Nvarchar(50),@pRowCount  Nvarchar(50) AS
BEGIN 
SET NOCOUNT ON
/******************************************************************************
DESCRIPTION:  <Generate Schema for Mockaroo https://www.mockaroo.com/ to generate Test data>
AUTHOR: 
PARAMETERS:  
 - INPUT(S)

- OUTPUT(S) 
   null
RESULT SET:
   null

select * from sys.types order by 1
select * from sys.columns order by 1
is_identity
0

MODIFICATION HISTORY: 
Examples: 
[tst].[sp_GenerateMockarooSchema] 'tbl_Main_Procurement','10'
[tst].[sp_GenerateMockarooSchema] 'tbl_Procurement','10'
-------------------------------------------------------------------------------
VERSION  DATE NAME   MODIFICATION
-------------------------------------------------------------------------------
*****************************************************************************/
DECLARE @columnMockarooSchemasStr varchar(max)=''
DECLARE @MockarooSchemasHeaderStr varchar(max)=''
DECLARE @MockarooSchemasFooterStr varchar(max)=''

DECLARE @TableHeaderStr varchar(max)=''
SET @TableHeaderStr=STUFF((
select ','+cast(col.name as VARCHAR(100))
from sys.columns col
left join sys.types typ
on col.user_type_id=typ.user_type_id
left join [dbo].[MockarooSchemas] MS
on typ.name=MS.[SQLServerDBType]
 where OBJECT_NAME(object_Id)=@pTableName
 AND col.is_identity=0  
 FOR XML PATH('')),1,1,'')

PRINT '--------Table Header Start---------'
PRINT ''
PRINT CAST(@TableHeaderStr AS NTEXT)
PRINT ''
PRINT '--------Table Header Start---------'
PRINT ''
PRINT ''

select col.name as ColumnName,'Column'+cast(row_number() OVer(order by col.column_id) as varchar(10)) AS ColumnNameAlias,typ.name as typeName,col.max_length,col.Precision,col.scale,col.is_nullable
from sys.columns col
left join sys.types typ
on col.user_type_id=typ.user_type_id
left join [dbo].[MockarooSchemas] MS
on typ.name=MS.[SQLServerDBType]
 where OBJECT_NAME(object_Id)=@pTableName
 AND col.is_identity=0


 SET @MockarooSchemasHeaderStr='{
  "num_rows": '+@pRowCount+',
  "file_format": "csv",
  "name": "'+@pTableName+'",
  "include_header": true,
  "columns": [
'
SET @MockarooSchemasFooterStr=']
}'

SELECT @columnMockarooSchemasStr=stuff((
SELECT

','+CASE WHEN typeName in ('bigint','decimal','float','int','numeric','tinyint','real','smallint','smallmoney','money') 
THEN Replace(Replace([MockarooSchemaDef],'<ColumnName>',ColumnNameAlias),'<Scale>',scale)
WHEN typeName in ('date') 
THEN Replace(Replace([MockarooSchemaDef],'<ColumnName>',ColumnNameAlias),'<Scale>',scale)
WHEN typeName in ('datetime','datetime2','datetimeoffset','time','timestamp','smalldatetime') 
THEN Replace([MockarooSchemaDef],'<ColumnName>',ColumnNameAlias)
WHEN typeName in ('bit') 
THEN Replace([MockarooSchemaDef],'<ColumnName>',ColumnNameAlias)
WHEN typeName in ('char','nchar','ntext','text','varchar','nvarchar') 
THEN Replace(Replace([MockarooSchemaDef],'<ColumnName>',ColumnNameAlias),'<MaxLength>',max_length/2)
WHEN typeName in ('geography','geometry','hierarchyid','image','varbinary','binary','xml','uniqueidentifier','sql_variant','sysname') 
THEN NULL END 
FROM(
select col.name as ColumnName,typ.name as typeName,col.max_length,col.Precision,col.scale,col.is_nullable,
 case when col.is_nullable=1 THEN replace(MS.[MockarooSchemaDef],'<NullPercentage>','30') ELSE replace(MS.[MockarooSchemaDef],'<NullPercentage>','0') END as [MockarooSchemaDef],
'Column'+cast(row_number() OVer(order by col.column_id) as varchar(10)) AS ColumnNameAlias
from sys.columns col
left join sys.types typ
on col.user_type_id=typ.user_type_id
left join [dbo].[MockarooSchemas] MS
on typ.name=MS.[SQLServerDBType]
 where OBJECT_NAME(object_Id)=@pTableName
 AND col.is_identity=0
) M
FOR XML PATH('')),1,1,'')

PRINT '--------MockarooSchema Definition Start---------'
PRINT ''
PRINT CAST(@MockarooSchemasHeaderStr+@columnMockarooSchemasStr+@MockarooSchemasFooterStr AS NTEXT)
PRINT ''
PRINT '--------MockarooSchema Definition End---------'

END
GO
