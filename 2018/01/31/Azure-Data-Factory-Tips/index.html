Azure Data Factory Tips

Posted on 2018-01-31

Load file by using wildcard file name

when you would like to get the files by using the file name pattern, like ‘File20180101123311.txt’.
Here is the tips:

Azure Blob Storage as input dataset

"typeProperties": {
    "folderPath": "adftutorial/File{Year}{Month}{Day}",
    "format": {
        "type": "TextFormat",
        "columnDelimiter": ","
    },
    "partitionedBy": [
        {
            "name": "Year",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "yyyy"
            }
        },
        {
            "name": "Month",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "MM"
            }
        },
        {
            "name": "Day",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "dd"
            }
        }
    ]
},

Azure Datalake store / On Promise File / SFTP as input dataset

Tested with Azure Data Factory version 1.0, when the input dataset has the folder structure, we cannot using the way we used for Azure Blob Storage.

Also the Property fileFilter for On Promise File / SFTP as input dataset cannot use the system variable like {Year},{Month} as its value.

But we can match the filename like ‘File20180101.txt’

"typeProperties": {
    "folderPath": "adftutorial/{Year}/",
    "fileName": "File{Year}{Month}{Day}.txt",
    "format": {
        "type": "TextFormat",
        "columnDelimiter": ","
    },
    "partitionedBy": [
        {
            "name": "Year",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "yyyy"
            }
        },
        {
            "name": "Month",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "MM"
            }
        },
        {
            "name": "Day",
            "value": {
                "type": "DateTime",
                "date": "SliceStart",
                "format": "dd"
            }
        }
    ]
},

refer to: https://social.msdn.microsoft.com/Forums/en-US/bacd7d7e-bd40-490c-b2c8-4fbcda67944d/dataset-dynamically-set-filename-by-offsetting-date?forum=AzureDataFactory

Copy Activity

Copy activity will only take the first input dataset as the source data

In the file text format, \r\n cannot be recognized, remove the row delimiter will use default[\r\n,\r,\n]

The table type sqlWriterProcedureParameter’s parameter name need to be same as the tablename property of the input dataset

Data Factory Column Mapping

Azure SQL Database Source Dataset can have 4 columns in the structure, but the source database table has 6 columns. Azure SQL Database Source Dataset structure map with source database table column by the column name, so the column order in structure doesn’t matter.

Column mapping supports the following scenarios:

  • All columns in the source dataset structure are mapped to all columns in the sink dataset structure.
  • A subset of the columns in the source dataset structure is mapped to all columns in the sink dataset structure.

The following are error conditions that result in an exception:

  • Either fewer columns or more columns in the “structure” of sink table than specified in the mapping.
  • Duplicate mapping.
  • SQL query result does not have a column name that is specified in the mapping.

From https://docs.microsoft.com/en-us/azure/data-factory/data-factory-map-columns

Copy Data From Access DB through ODBC

– We need grant access fullcontrol on below registry HKEY_LOCAL_MACHINE\SOFTWARE\ODBC
– Also grant access fullcontrol on the folder where the Access DB file located.

Link Service as below: 

{
    "$schema": "http://datafactories.schema.management.azure.com/schemas/2015-09-01/Microsoft.DataFactory.LinkedService.json",
    "name": "LS_Name",
    "properties": {
        "type": "OnPremisesOdbc",
        "typeProperties": {
            "connectionString": "Driver={Microsoft Access Driver (*.mdb, *.accdb)};Dbq=C:\\folder\\aa.mdb;",
            "gatewayName": "dmg01",
            "authenticationType": "Anonymous",
            "credential": ""
        }
    }
}

Cannot Rerun the activity from Data Factory Monitor

Sometime we cannot rerun the activity from the Data Factory Monitor, we click rerun, then refresh the page, the activity status is still showing “Waiting”. It looks like a bug on the data factory monitor.

In this case, we can go back to the Data Factory home page, go to Contents->Pipelines, select the pipeline and check the activity dataset,
drill down to the status of the slice of the dataset, then click run or valid to verify the status. it will start the actvity.
